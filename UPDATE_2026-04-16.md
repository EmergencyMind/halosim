# HaloSim — Project Update
**Date:** April 16, 2026  
**Status:** Initial build complete — ready to deploy

---

## What we built

HaloSim is a generalization of the "Code Blue Blindspots" paper methodology into a free, shareable, web-based simulation tool. Anyone can use it to model how often their providers actually encounter HALO events — and whether training programs are making a meaningful difference.

The app lives at `projects/halosim/` as its own standalone git repository, separate from the rest of the sangfroid_labs workspace. When pushed to a public GitHub repo, it deploys to Streamlit Community Cloud in about two minutes with no configuration beyond clicking a button.

---

## What it does

**Feature 1 — Exposure modeling**

You define a provider population (up to 5,000), assign them work schedules, and either generate synthetic HALO events via a Poisson process or upload your own real event timestamps. The app then computes, for each provider, how many events they witnessed and how long the gaps between those events were — matching the specific methodology from the paper (shift-typed exposure: day events go to day-shift providers, night events to night-shift providers).

The primary output is a distribution of inter-exposure gaps across the population, plus a threshold sweep chart showing what percentage of providers go more than X days without a live exposure — at any threshold you choose.

**Feature 2 — Training simulation**

You add a training program (monthly, bi-monthly, quarterly, custom interval, or targeted) and the app runs a second simulation to model how that training changes population readiness over time. The key output is a time series of what fraction of on-shift providers are "ready" on any given day — with training versus without.

"Readiness" in Basic mode is a binary threshold: you're ready if you've had a live exposure or training event within the last N days (default 90). Advanced mode adds exponential decay, an Ebbinghaus forgetting curve, or a two-threshold step model.

---

## Key design choices worth knowing

**Basic / Advanced mode.** The sidebar has a toggle. Basic mode uses the binary threshold model, simple Poisson events, and full-reset training — appropriate for operational questions. Advanced mode unlocks separate day/night event rates, seasonal variation, custom 28-day schedule patterns, CSV/Excel upload for both events and schedules, all four readiness decay models, partial training boost, and targeted training (which trains only providers whose readiness has fallen below a threshold). Switching modes doesn't reset your inputs.

**Readiness is measured on-shift only.** The proportion_ready metric only counts providers who are currently working — a provider off-shift can't respond to a HALO event regardless of their readiness score. Days where nobody is on shift return NaN, not zero. This was a design flaw in the original plan that the red team caught and the implementation fixes.

**Targeted training is slow by design.** Unlike the other training programs (which are instant matrix operations), targeted training runs a day-by-day loop because each day's decision about who to train depends on that day's readiness state. For 1,000 providers over 365 days it's fast enough; for 5,000 × 730 it will take a few seconds. This is correct behavior — it's an agent-based simulation.

**The 98% figure from the paper won't reproduce exactly.** The paper's result — that 98% of nurses went more than 90 days without a cardiac arrest — came from 208 real events with actual temporal clustering. A Poisson process at the same average rate will produce something in the ballpark (this build showed 28% at the same rate but with only 200 providers and 365 days), but won't match exactly. To reproduce the paper's result, upload the actual event data.

## Next steps to go live

1. Create a public GitHub repo (e.g. `github.com/ddworkis/halosim` or similar)
2. From `projects/halosim/`:
   ```bash
   git remote add origin git@github.com:<you>/halosim.git
   git push -u origin master
   ```
3. Go to [share.streamlit.io](https://share.streamlit.io), sign in with GitHub, click New App, select the repo, set main file to `app.py`, deploy
4. Update the citation URL in `README.md` with the live URL

That's it — no server, no cost, publicly accessible.
